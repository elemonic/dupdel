# Changelog

## 2026-04-08 サイズ集計と verbose 表示を調整

- 通常のファイル重複モードでも、削除予定サイズ合計、実削除サイズ合計、削除失敗サイズ合計を bytes で集計・表示するようにした。
- dry-run 時も削除予定サイズ合計を画面表示するようにした。
- 複数フォルダ処理時、通常時は重複なしフォルダのフォルダ名と `(重複なし)` を画面表示しないようにした。
- `--verbose` / `-v` を追加し、指定時のみ重複なしフォルダも画面表示するようにした。
- verbose はコンソール表示だけに作用し、ログ出力の詳細度は従来どおりにした。
- help と README.md に `--verbose` / `-v` とサイズ集計の説明を追記した。

## 2026-04-08 Windows コマンドライン引数の日本語パス処理を修正

- Windows では `@ARGV` を cp932 から Perl 内部文字列へ正規化してから解析するようにし、コマンドプロンプトで日本語を含む対象フォルダを指定した場合に存在確認が失敗する問題を修正した。
- すでに mojibake 状態で UTF-8 フラグが付いている引数にも対応できるよう、latin-1 バイト列へ戻して cp932 デコードを試みる補正を追加した。

## 2026-04-07 ハッシュ方式選択オプションを追加

- `--hash ALG` と `-H ALG` を追加し、重複判定に使うハッシュ方式を選べるようにした。
- ハッシュ方式のデフォルトを `sha256` に変更した。
- 指定可能な方式を `sha1` / `sha256` / `blake2` / `blake3` に制限し、未対応の値はエラーにするようにした。
- `sha1` / `sha256` は `Digest::SHA`、`blake2` は CryptX の `Crypt::Digest`、`blake3` は `Digest::BLAKE3` を選択時に使うようにした。
- `blake2` / `blake3` の必要モジュールが利用できない場合は、別方式へフォールバックせず分かりやすいエラーで終了するようにした。
- 実行時表示、`--delete` 時ログヘッダ、help、README.md にハッシュ方式の情報を追加した。

## 2026-04-07 フォルダ重複削除オプションを追加

- `-F` を追加。対象フォルダ直下の子フォルダ同士を比較し、直下ファイルの名前集合と内容が完全一致する重複フォルダを検出するようにした。
- サブフォルダを含む子フォルダは比較対象外としてスキップし、画面とログにスキップ理由を出すようにした。
- `--delete` 時は delete 側フォルダ内の直下ファイルを先に削除し、実際に空になった場合のみフォルダ自体を削除するようにした。
- `-e REGEX` にマッチするファイルを含む重複フォルダグループは、フォルダ削除全体をスキップするようにした。
- フォルダ重複削除モード向けに、削除予定サイズ合計、実削除サイズ合計、削除失敗サイズ合計を bytes で表示・ログ出力するようにした。
- help と README.md に `-F` の使い方と注意点を追記した。

## 2026-04-07 削除除外 regex オプションを追加

- `-e REGEX` を追加。ファイル名が `REGEX` にマッチするファイルを削除候補から除外するようにした。
- 除外判定はフルパスではなくファイル名だけに対して行い、除外にマッチしたファイルは重複していても削除しない仕様にした。
- 除外対象を delete 禁止として扱い、除外対象が複数ある場合に重複が複数残る挙動を許容する形にした。
- 実行時表示、`--delete` 時ログヘッダ、help に削除除外 regex の情報を追加した。
- README.md を追加し、現在の使い方、オプション、削除除外 regex の扱いを記載した。

## 2026-04-01 事前ベースライン

- changelog を初作成。
- `dupdel.pl` は基準フォルダ直下の通常ファイルだけを対象にする単一フォルダ処理。
- 重複判定はサイズ絞り込み後の SHA-1 ベース。
- keep はファイル名の文字列比較で最小の 1 件、残りを削除候補にする仕様。
- `--delete` 時のみ `yyyymmdd-DUPDEL.log` に UTF-8 追記ログを出力。
- コンソール表示は `WinConsoleJP.pm` 経由で Windows cp932 を考慮し、ログは UTF-8 を保持する構成。
- Windows では `Win32::LongPath` を優先し、非 Windows では core 関数へフォールバックする構造。

## 2026-04-01 サブフォルダ単位処理を追加

- `-d N` を追加。基準フォルダから見てちょうど `N` 階層下の各フォルダを独立に処理。
- `-D N` を追加。1 階層下から `N` 階層下までの各フォルダを独立に処理。
- `-s` を追加。基準フォルダ自身も処理対象へ含める指定を追加。
- 対象フォルダ集合の決定と、1 フォルダ内の重複検出処理を分離。
- 複数フォルダ処理時は画面表示とログにフォルダ単位の区切りを追加。
- ログに各処理フォルダの件数サマリと、実行全体の合計サマリを追加。
- Windows の `Win32::LongPath` を使う列挙・open・unlink 経路は維持。

## 2026-04-01 特殊値 `0` を追加

- `-d 0` を追加。下位フォルダを持たない葉フォルダのみを処理対象にする仕様を追加。
- `-D 0` を追加。基準フォルダ配下の全サブフォルダを処理対象にする仕様を追加。
- help 文言と処理単位表示を特殊値の意味に合わせて更新。

## 2026-04-03 優先削除文字列と削除順オプションを追加

- `-p STRING` を追加。重複グループ内で `STRING` を含むファイルを削除側へ寄せる keep 選択ルールを追加。
- `-r` を追加。ファイル名の逆順で keep を決めるオプションを追加。
- keep/delete を決める並び替えロジックを関数化し、優先削除文字列と逆順指定を組み合わせ可能にした。
- 実行時表示とログヘッダに keep 選択ルールを追加。
- help 文言を新オプションに合わせて更新。

## 2026-04-05 削除失敗時の詳細ログを追加

- 削除失敗時の通常ログ記録は維持したまま、keep/delete の対応が分かる失敗情報を内部で蓄積するようにした。
- ログ末尾に `削除失敗一覧` セクションを追加し、`TARGET_DIR`、`KEEP`、`DELETE`、`ERROR`、`TIME` をまとめて出力するようにした。
- コンソール警告にも keep/delete の対応を含め、発生時点でどの重複グループの失敗か分かりやすくした。
